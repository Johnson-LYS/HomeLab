# 安全护栏（guardrails）

> 约束**一切** AI 运维操作。动手前先读本文件（见 `AGENTS.md` §3）。
> 本文件是 **A→C 的调节旋钮**：把操作在四级间移动即可调整自动化程度。**红线永不晋升。**

## 当前阶段：A + 辅助执行

- 自动放行：仅**只读**类。
- 其余改动：AI 出方案+命令，**人确认/执行**，AI 写 `operations/log`，**在人主动明确授权的情况下，AI 可以直接执行，但是危险操作需要再次确认**。

---

## 🔴 红线（RED）— 未经人工明确批准，绝不执行

| id     | 规则                                                                   | 原因                                 |
| ------ | ---------------------------------------------------------------------- | ------------------------------------ |
| RED-01 | 不格式化 / 删除 / 重建 QNAP 那块 **16TB 唯一盘** 的任何卷或数据        | 单盘无冗余，唯一副本，丢即永久丢失   |
| RED-02 | 不修改 **Mac mini Surge 网关** 配置 / 不停 Surge                       | VM 网关模式，一动全家断翻墙          |
| RED-03 | 不改 **主路由(中兴星云 MAX)** 拨号/WAN/防火墙/端口转发，不改 mesh 拓扑 | 可能全家断网或把自己锁在外面         |
| RED-04 | 不改 **AdGuardHome 上游/核心解析** 导致全网 DNS 失效                   | .15 的 DNS 服务全网依赖              |
| RED-05 | 不删除 / 轮换 **v2fly·vmess、DNS API token** 而不先确认影响            | 高危：入内网通道 / 域名控制权        |
| RED-06 | 任何 `rm -rf`、磁盘分区、`mkfs`、`dd`、批量删容器/卷                   | 破坏性不可逆                         |
| RED-07 | 不向被 git 跟踪的文件写入任何明文密钥                                  | 进 git 历史 + 进 AI 上下文，永久泄露 |

## 🟠 前置条件（PRECONDITION）— 满足后方可进行

| id     | 操作                               | 必须先做                                   |
| ------ | ---------------------------------- | ------------------------------------------ |
| PRE-01 | 改 PVE 上任何 VM/CT                | 先打**快照**                               |
| PRE-02 | 改 NPM / AdGuardHome / 1Panel 配置 | 先**备份**当前配置（导出/复制）            |
| PRE-03 | 任何网络变更（IP/路由/DNS/防火墙） | 先确认**还有另一条路**能进设备（不会自锁） |
| PRE-04 | 重启/停止服务                      | 先确认依赖方影响，记录回滚命令             |

## 🟡 需人工确认（CONFIRM）— AI 给方案，人点头后执行

- 重启 / 停止 / 部署 / 更新任何服务或容器。
- 修改任何配置文件。
- 一切涉及 **P0 公网暴露面**（1Panel/PVE/QNAP/Emby/NPM/ddns-go/vmess）的写操作。
- 安装 / 卸载软件包。

## 🟢 自动放行（AUTO）— AI 可直接做

- 只读诊断：`status`、`ps`、`df`、`free`、`uptime`、SMART 读取、日志查看。
- 只读 API 查询（GET 类）。
- 读取 inventory / operations，生成报告与建议。
- 起草（但不提交执行）命令与变更方案。

## 晋升规则（A→B→C）

- 某操作在 `operations/log` 中被 AI **成功执行 ≥N 次、无意外**（建议 N≥3）→ 可从 CONFIRM 下移到 AUTO。
- 下移须：① 在此文件记录晋升（含依据 log 链接）；② 同步到 `.claude/settings.json` 机械护栏。
- **红线（RED）任何情况下不下移。**

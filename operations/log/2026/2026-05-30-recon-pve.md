# 只读盘点 PVE（极摩客 .16）

## 目的
摸清虚拟化实验平台 PVE 的版本、VM/LXC、存储与跨机依赖。

## 执行
只读，`ssh root@192.168.8.16`（无别名，直连；1Password Agent）：
```
pveversion; qm list; pct list; pvesm status; lscpu; free -h; lsblk; pvecm status
```

结果：
- PVE 9.1.1 / kernel 6.17 / 主机名 `pve` / uptime 153 天 / 单节点。
- 硬件：Intel N100 4核、15Gi 内存、512G INTEL nvme（与 inventory 一致）。
- 12 个 VM（多 stopped 实验），唯一运行：**102 fnOS**（虚拟 NAS）；1 个 LXC moltbot(stopped)。
- 存储：local(dir)/local-lvm(lvmthin) + **qnap_ssd(NFS，挂自 QNAP)**。

## 验证
qm/pct/pvesm 命令均正常返回（root 权限足够），数据自洽。

## 关键发现
- **跨机依赖 PVE→QNAP**：qnap_ssd 为 NFS，QNAP 宕机会波及 PVE 上用此存储的 VM（含常开 fnOS）。已记入 services.md 依赖关系。
- 家里其实有**两个 NAS**：物理 QNAP + 虚拟 fnOS(VM102)。

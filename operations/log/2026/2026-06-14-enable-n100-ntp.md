# 启用 n100 内网 NTP 服务

## 目的
摄像头为阻止联网配置了无效网关，无法访问外部 NTP，导致画面 OSD 时间漂移。为保持摄像头不出网，同时可校时，在 n100 提供内网 NTP 服务。

## 前置
- 已只读确认 n100 宿主机时间正常：`Asia/Shanghai`，NTP 已同步。
- 已确认 n100 原本没有监听 UDP/123，`systemd-timesyncd` 仅作为 NTP client。
- 系统级 `chrony` 安装被 sudo 密码阻止；1Password CLI 当时无法连接桌面 App，未获取任何密码。

## 执行
采用 Docker 方式部署，避免写入宿主系统包状态：

- 构建目录：`/home/johnson/homelab-ntp`
- 基础镜像：`alpine@sha256:56fa17d2a7e7f168a043a2712e63aed1f8543aeafdcee47c58dcffe38ed51099`
- 安装包：`chrony 4.6.1-r0`
- 本地镜像：`homelab/chrony-ntp:20260614`
- 容器：`homelab-ntp`
- 启动参数：`--network host --restart unless-stopped --read-only`
- chronyd 参数：`-d -x`，禁用容器对宿主机系统时钟的控制。

chrony 配置要点：

```text
bindaddress 192.168.8.15
allow 192.168.8.0/24
local stratum 10
cmdport 0
```

## 验证
- 容器 `homelab-ntp` 已运行，restart policy 为 `unless-stopped`。
- n100 已监听 `192.168.8.15:123/udp`。
- 从 n100 主机发 NTP 查询到 `192.168.8.15:123` 成功：
  - `stratum 4`
  - server UTC: `2026-06-14T09:04:14.422777+00:00`
  - host local: `2026-06-14T17:04:14.425224+08:00`
  - delta: about `-0.003s`

## 摄像头后续配置
- 摄像头 NTP server 指向 `192.168.8.15`。
- 摄像头时区设为 `GMT+08:00` / `Asia/Shanghai` / `Beijing`，DST 关闭。
- 主路由防火墙建议：允许 `192.168.8.50-192.168.8.59 -> 192.168.8.15`，再丢弃 `192.168.8.50-192.168.8.59 -> 0.0.0.0-255.255.255.255`。

```bash
docker image rm homelab/chrony-ntp:20260614
rm -rf /home/johnson/homelab-ntp
```
